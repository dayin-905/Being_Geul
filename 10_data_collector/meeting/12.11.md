## 1. 🎨 Frontend (UI/UX 및 로직 구현)

### 1) 웹페이지 로딩 & 진입 플로우
* **로딩 화면:** 검정색 이모티콘 스와이프 로딩 (SVG) 적용.
* **진입 순서:** `로딩` → `로그인(팝업) or 둘러보기` → `지도(지역 선택)`.
* **지도 UI 구조:**
  * **1단계:** 9개 권역으로 나누어진 대형 지도 제공.
  * **2단계:** 권역 클릭 시 그 안의 상세 구역(총 19개 지역)이 보이는 상세 지도 제공.
  * *Note: 상세 지도 이미지 제작 업무 일부는 PM이 분담.*

### 2) 메인 및 탭 구성
* **알고리즘 탭:** 현재 슬라이딩 방식 유지, 추천 정책 위주 노출.
* **모든 정책 보기 탭 (New):**
  * 탭 명칭: **'New' 탭** (가칭).
  * UI 형태: **그리드(Grid)** 형.
  * 기능 이동: **검색창(Search Bar)**을 이 탭으로 이동하여 배치.
* **정렬 기능 (Sorting):** 최신순 / 상시 / 마감임박 솔팅 바 제공 **[확정]**.

### 3) My Page (마이페이지)
* **내 정보 수정:** 프로필 카드 내 `[내 정보 수정]` 탭 활성화 시 **팝업창**으로 구현.
* **활동 지수 정의:** 찜한 목록 중 **'상세 보기를 클릭하지 않은(안 읽은)'** 정책 개수로 정의.
* **디데이(D-Day):** 내가 찜한 정책 카드 내부에 D-Day 표기 **[확정]**.
* **원문 보기 탭:** 인스타그램의 '좋아요/저장 컬렉션' UI 참고하여 별도 탭 구현 **[확정]**.

## 2. 🛠️ Backend & Data (데이터 및 로직)

### 1) 데이터 노출 로직
* **지역 선택 시:** `[선택한 지역 정책]` + `[전국구 정책]` 합산 노출.
  * *순서 중요, 추후 최신순/마감임박순 정렬 로직 추가 예정.*

### 2) 데이터 분류 및 관리
* **[지호님]**:
  * DB를 지역별(전국+제주 / 광역시 등)로 나누어 개수 파악 시트 제공.
  * 효율적인 지역 구분 방안 제시 예정.
* **[영후님]**:
  * 프론트엔드 HTML + 백엔드 DB 연동(Merge) 작업 착수.

### 3) '깡통 데이터'(상시/기간 포괄) 처리 정책
* **이슈:** 마감일이 '1월 1일~12월 31일'로 뜨거나 기간이 '상시'인 데이터 처리.
* **판단:** 2030 청년에게 정보 제공이라는 목적에 부합하므로 **삭제하지 않음**.
* **결정:** 해당 데이터의 일자를 **"상시"**로 분류하여 제공 **[확정]**.

### 4) 기타
* **일반화 프롬프트:** 황욱님께 업무 배당 (프롬프트 작성 및 테스트).
