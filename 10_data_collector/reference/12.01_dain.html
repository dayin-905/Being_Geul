3. 상세 기능 명세서 (Functional Specs)

3-0. Intro: 다이내믹 로딩 (INTRO-00)

기획 의도 "텍스트들이 제자리를 찾아간다"는 팀 아이덴티티 표현 및 시각적 재미(Fun) 전달.
시나리오 1. Chaos: 정책 키워드들(#주거, #100만원 등)이 화면에서 무질서하게 회전/부유.
2. Order: "빙글!" 느낌과 함께 중앙으로 빨려 들어가며 로고로 변신(Morphing).
3. Welcome: "당신을 위한 정책을 찾았어요!" 문구 출력 후 섹션 1 등장.
Tech Spec - GSAP Timeline: 순차적 애니메이션 제어.
- SessionStorage: 재방문 시 애니메이션 생략 여부 체크.
- Pre-loading: 애니메이션 재생 중 백그라운드에서 API 데이터 호출.

3-1. Section 1: 정책 매칭 (MAIN-01)

UI 구성 화면 중앙 카드 덱(Card Deck). 텍스트 위주(제목, 핵심요약, D-Day) 디자인.
스와이프 로직 - 왼쪽 드래그: 좋아요 (Like) → 저장 및 분석 반영.
- 오른쪽 드래그: 패스 (Pass) → 다음 카드.
* 카드 소진 시 Section 4(탐색) 이동 버튼 노출.
클릭 인터랙션 카드 덱 최상단 클릭 시 [상세 정보 모달] 호출.
Tech Note
Event Conflict: 드래그와 클릭을 구분하기 위해 이동 거리가 5px 미만일 때만 'Click'으로 간주하는 Threshold 로직 필수 구현.

3-2. Section 2: 취향 분석 (ANAL-01)

기능 정의 사용자가 '좋아요'한 정책의 태그 비중(%)을 시각화.
UI/Interaction - Floating Bubbles: 키워드 버블이 둥둥 떠다니는 무중력 효과.
- 선호도가 높을수록 버블 크기가 크고 움직임이 묵직함.
- 마우스 오버 시 일시 정지 또는 확대.

3-3. Section 3: 나의 보관함 (SAVE-01)

UI 구성 높이가 다른 카드들이 벽돌처럼 쌓이는 Masonry Layout (핀터레스트 스타일).
기능 - 카드 클릭 시 상세 모달 호출.
- '하트' 아이콘 클릭 시 삭제 (삭제 후 'Undo' 토스트 팝업 제공).

3-4. Section 4: 정책 탐색 (EXPL-01)

검색/필터 - 상단 검색바 (실시간/엔터).
- 가로 스크롤(Horizontal Scroll) 가능한 카테고리 태그 필터.
리스트 전체 DB 조회. 저장되지 않은 항목은 빈 하트로 표시.

3-5. Footer (FOOTER-01)

위치/구성 - 최하단 고정 (Static Component).
- Contents: 팀 로고(빙글), 슬로건, 저작권 표기(© 2025), 팀 소개/문의 링크.

상세 정보 모달 (MODAL-01)

형태 배경 Dim 처리된 오버레이 팝업 (페이지 이동 없음).
